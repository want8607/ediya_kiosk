==수업==

1. 4대 컴포넌트(Activity, Service, Broadcast Receiver, Conetents Provider)
	컴포넌트 : 독립적으로, 혼자 동작할 수 있는 단위 (context)
	Service : 쓰레드
	Broadcast Receiver : 특정앱에 알림을 보내기(게임에서 목표과제 달성)
	Contents Provider: 앱들 사이에 데이터 전송

2.생명주기
Activity의 생명주기

-onCreate : 앱 아이콘을 누름, 레이아웃 연결, 버튼 이벤트 등록	
-onStart : 액티비티가 화면에 보이는 과정, onStop에서 설정한 값을 다시 넣어줌
-onResume : 유저의 입력을 받을 수 있는 상태, onPause에서 설정한 값을 다시 넣어줌
<<앱 구동 중 >>
-onPause : 액티비티의 일부가 가려졌을 때 나오는 상태
-onStop : 액티비티의 전체가 가려졌을 때 나오는 상태 (액티비티 전환, 다른 앱 실행)
-onDestroy : 앱 종료, 뒤로가기 눌렀을 때만 실행

액티비티 개수가 적으니까 stop되었을 때 초기로 돌아가는 걸 고려해야함


특징
1. 안드로이드의 경우, 한화면에, 한앱의 한 페이지만 띄울 수 있음
2. 내 핸드폰에 현재 켜져있는 앱이 뭔지 확인을 하지 않음
3. 앱들 중에, 화면을 차지하는 것만 활성화 되어있다고 볼 수 있음
